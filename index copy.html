The World Wide Web

The web was created in the early 1990s and is estimated to be 30 billion pages. The main components of the web are the Internet, a browser, servers, and HTTP. HTTP is the main protocol of the web, and servers are computers that host files on the web. The Internet is the World's largest computer network, and a browser is a program that can display files that are found on the web. The browser, such as Firefox. Chrome, Safari, Edge, or Microsoft Explorer, makes requests via the HTTP protocol to display / render pages that are hosted on servers.

The Basics of HTML

Introduction to HTML Tags

HTML is text content (what you see), markup (what it looks like), and any references to images, videos, and links to other pages. To make HTML look different, tags are used. The first tag is called the opening tag and consists of the tag name enclosed by a less than symbol (<) and a greater than symbol (>). Then there are the contents, and on the other side is the closing tag, which is made up of a less than symbol, a slash (/), the tag name, and a greater than symbol. The whole structure is referred to as an element. So, for example, a header one element will look something like this: <h1>Your Header Here</h1>. There are other types of tags such as the <b> tag, which can bold text, and the <em> tag, which will display text in italics.

Advanced HTML Tags

An HTML tag can contain an attribute. Attributes always equal a value. Additionally, tags can have multiple attributes. An example of a tag that uses an attribute is called the anchor tag. The anchor tag <a> can link a word or a phrase to a webpage such as this: Udacity. Displayed in a browser, the word "Udacity" is linked to the Udacity homepage.

Another advanced HTML tag is the image tag. The img tag structure includes the source for the image in the form of an URL or a reference to a file location. There is also an alt component of the tag that includes the text that is displayed when the image doesn't load. So, if the URL doesn't exist, the text will display instead. Also, this type of tag is called a void tag and has no ending slash. It is void because it has no content.

There are a lot of details to remember about HTML. Fortunately, many resources exist on the web. Some of these include the Mozilla Developer Network and their section on HTML, and W3Schools, where one can find plenty of documents on web development and HTML tags.

HTML Page Structure

In HTML, text on multiples lines will render on a single line. In order to use multiple lines, there must be a line break. A line break is acheived by using the break tag, and is a void tag. Another way to achieve a line break is by using the paragraph tag. The paragraph tag is not considered a void tag because it has content. When using the paragraph tag, there will be a space between each paragraph.

Inline vs. Block Elements

When writing text in HTML, there can be as many spaces as you'd like between the words, but it will always render as a single space between each word. The sames goes for text that is broken up over several lines; it will still render on a single line. The break tag <br> is an inline tag. It ends a line and everything that comes after will wrap to the next line. For example:

Here is one line...
And here is another line...

On the other hand, the paragraph tag, which also forces a line break, creates an invisible box that has height and width. Some examples of inline elements are the bold tag, the emphasis tag, and the image tag.

Two additional elements are the span tag and div tag. The span element creates inline container, whereas the div element creates a box container. Attributes can be added to each of these elements. A type of attribute that can be added to each is class, which is CSS. CSS dictates the style of the resulting HTML page. The inline elments include span, a, br, img, and strong. Some block elements are div, p, and form.

HTML Documents

HTML documents can be quite complex in their markup. The head section, which contains the title, can also include meta-data, JavaScript, and CSS. The body tag contains the actual content of the page. The majority of this course will focus on what is between the body tags.

CSS Style

The Basics of CSS

CSS (Cascading Style Sheets) is used to style HTML. Using the following analogy, HTML can be thought of as the structure of a house, such as its walls. Style would then be the color of the paint used on the walls or the type of flooring used throughout the house. Additionally, JavaScript is the functionality of a webpage, so using the house analogy, it would be something like a garage door opener or a remote control for the television.

When writing CSS, a Selector must first be chosen. Basically, when a selector is indicated in the code, all of the rules written for it will be applied in the HTML document. So, if h1 is chosen as the selector and the CSS rules indicate that the background color is black and the text color is blue, this will apply to all h1 elements.

Here are some important CSS definitions:
  • The Rule is the CSS code that describes the value an attribute should take.
  • The Property is what you want to change.
  • The Value is what is assigned to the attribute.
  • The Selector is a name chosen by the programmer to target elements that are assigned to an id attribute or a class in HTML.
  • The Class is a group of elements that can be styled together. When using Class names, there should be no periods or any other punctuation marks.
  • The ID is a unique identifier for an element in the HTML document.

Advanced CSS Topics

In the example above, the CSS rule was applied to every header in the HTML doc. what do we do if we only want to apply certain rules to certain h1's? To answer this question, class selectors must be used. When a class is selected in HTML, it allows the programmer to specifically target that class, or portion of the HTML doc, for specific CSS styling rules. This allows for greater flexibility in styling a HTML document.

There are three ways that CSS can be added to HTML:
  1. Add a <head> element and include a <style> element. In the <style> element, write the CSS code for the document.
  2. Write the CSS code inline with the HTML. This is regarded as the worst option. It is bad because the programmer will have to write a lot of repeated code, and when code is repeatedly written, it can lead to mistakes.
  3. Write a separate CSS file and link it to the HTML file. This is regarded as best method to include style because it allows the programmer to stay organized.

It is important to note that if a programmer choses to write a separate CSS file and link it to the HTML file, then both files (and any images) must be contained within the same master project folder. Lastly, check your HTML and CSS files with the HTML validator and the CSS validator to make sure the code is written correctly!

The Box Model

Everything on a webpage is composed of boxes. However, before these boxes can be positioned and/or re-shaped on a webpage, it is first helpful to understand the box model.

The box is made of up four components:
  1. Content: Image or text that appears on the webpage. The layers that surround it can be viewed as protection, so boxes do not sit too close to each other.
  2. Padding: Clears an area around the content.
  3. Border: Goes around the padding and content.
  4. Margin: Clears an area around the border. The margin does not have a background color. It is completely transparent because it exists as the space between boxes.

The size of the box is equal to the border, padding, and content widths. Additionally, box sizes can be set as pixels or as percentages. If the box size is set as a percentage, the box size will change depending on the size of the webpage. This will not happen if the box size is set in pixels. The box size will remain the same size as the webpage changes. This can lead to a bad user experience.

Moving boxes around is probably one of the most difficult tasks in CSS. One way this can be achieved is by using the flex box method. This is an efficient way to distribute space among the lines in a container, or div. Adding "display: flex" to the appropriate CSS will display divs next to each other.
